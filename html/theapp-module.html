chunk(text)

source code 

Parses a specifically determined chunk sequence and utilises it for chunking. It is then used to parse the text segment, provided as a parameter. Eternally outputs a GUI, including a graph of the chunk parser's output. The interface consists of a new window, per sentence.

Parameters:
  • text (string) - holds the text segment
Returns: void
graphical output

analyseFile(filename, a)

source code 

This is effectively the main, collective method of execution. It initially attempts to open the file, from the provided path/name. The file's content is split into an array, one element per line. Should the user have entered the full analysis parameter "-a", tokenisation, POS tagging, filtering, chunking and stemming are performed, in this order.

Parameters:
  • filename (string) - holds the name of the input file
Returns: void
file output
Raises:
  • err - the error occurs, should the file not be found

parseHTML(url, num)

source code 

Essentially downloads a web page, from a provided URL. Its HyperText Markup is then parsed, by means of BeautifulSoup. A regex is used to clean the web page's title, and assign the result to the output file. All lexicon tags from the file are then separated, and appended to a new string, to be returned.

Parameters:
  • url (url) - the web page's uniform resource locator
Returns: string
holds the majority of words within the original text
